## 双Agent对抗模型

### Agent Builder（功能验证 · 乐观派）

| 职责 | 验证所有功能按预期工作 |
|------|----------------------|
| 策略 | 模拟正常用户操作路径，验证 happy path |
| 覆盖 | 登录流程、首页渲染、TabBar导航、问卷流程、个人中心 |
| 判定 | 如果Builder失败 → 基础功能存在缺陷 |

### Agent Reviewer（对抗验证 · 悲观派）

| 职责 | 寻找边界条件和异常场景 |
|------|----------------------|
| 策略 | 模拟异常输入、网络故障、竞态条件、权限缺失 |
| 覆盖 | 安全性、未授权访问、异常数据、性能压力、竞态 |
| 判定 | 如果Reviewer通过 → 系统健壮性高 |

### 对抗裁定逻辑

| Builder | Reviewer | 裁定 | 行动 |
|---------|----------|------|------|
| PASS | PASS | HIGH_CONFIDENCE | 可以发布 |
| PASS | FAIL | REVIEW_NEEDED | 需要人工审查冲突 |
| FAIL | FAIL | CRITICAL | 确认的缺陷，立即修复 |
| FAIL | PASS | CRITICAL | 基础功能缺陷 |

## 环境准备

### 1. 安装依赖

```bash
cd tests
npm install
```

### 2. 配置开发者工具

1. 打开微信开发者工具
2. 进入 **设置 → 安全设置**
3. 开启 **服务端口** (CLI/HTTP 调用)

### 3. 修改配置（如需要）

编辑 `shared/config.js`：

```javascript
// macOS 默认路径
const CLI_PATH = '/Applications/wechatwebdevtools.app/Contents/MacOS/cli';

// Windows 路径示例
// const CLI_PATH = 'C:/Program Files (x86)/Tencent/微信web开发者工具/cli.bat';
```


## 测试覆盖矩阵

| 功能模块 | Builder测试 | Reviewer测试 |
|---------|------------|-------------|
| 登录页渲染 | ✓ 正确渲染验证 | ✓ 未勾选协议安全性 |
| 登录流程 | ✓ 正常登录 | ✓ 防重复点击、网络超时、空token |
| 首页Banner | ✓ 数据结构、渲染 | ✓ 空数据、null、500错误 |
| 首页旅程 | ✓ 有/无旅程展示 | ✓ 网络断开降级 |
| TabBar | ✓ 三个Tab切换 | ✓ 快速切换竞态 |
| 问卷Step1 | ✓ 选择、下一步 | ✓ 无效ID、快速连续点击 |
| 问卷Step2 | ✓ 饮食住宿选择 | ✓ undefined输入 |
| 问卷Step3 | ✓ 多选toggle | ✓ 提交失败保留数据 |
| 个人中心 | ✓ 信息展示、跳转 | ✓ 加载失败默认值、退出确认 |
| 页面栈 | - | ✓ 10层限制、reLaunch清空 |
| 大数据渲染 | - | ✓ 50条banner、100条eggs |
| 并发操作 | - | ✓ 并发setData |

## 扩展指南

### 添加新的Builder测试

在 `agent-builder/` 下创建新的 `.test.js` 文件，遵循模式：

```javascript
describe('[Builder] 新功能验证', () => {
  test('正常场景应该工作', async () => {
    // 模拟正常用户操作
    // 验证预期结果
  });
});
```

### 添加新的Reviewer测试

在 `agent-reviewer/` 下创建新的 `.test.js` 文件，遵循模式：

```javascript
describe('[Reviewer] 新功能对抗验证', () => {
  test('异常输入不应崩溃', async () => {
    // 注入异常条件
    // 验证系统不崩溃
  });
});
```

## 技术栈

- **miniprogram-automator**: 微信官方小程序自动化SDK
- **Jest**: 测试框架与断言库
- **Node.js 8+**: 运行环境
- **微信开发者工具**: 小程序运行容器
