# 更新日志

## 2026-06-27

### 新功能

- 行程生成算法引入"线路优先"逻辑：优先从同一线路的景点中选取，保证每天路线连贯
- 贪心最近邻路径排序：生成的景点自动按地理最短路径排序，减少来回折腾
- 盲游替换后二次排序：半盲模式下替换盲游景点后重新优化路径顺序
- 行程详情景点拖拽排序：长按500ms景点卡片可拖动调整顺序，松手自动保存
- 添加日程搜索改为API搜索：输入关键词实时调后端接口搜索（景点/住宿/餐饮/包车/跟拍）
- 盲游模式完整交互：
  - 景点卡片五态渲染（visible/unlocked/next/locked/skipped）
  - 地图光圈显示（fuzzy坐标 + 半径圈）
  - 景点解锁/跳过/强制解锁流程
  - GPS校验到达解锁
  - 全盲模式地图正确显示模糊坐标和虚线路径

### Bug修复

- 修复全盲模式下地图空白（真实坐标为null时直接return导致地图不渲染）
- 修复盲游地图神秘地点编号从02开始的问题（现从01开始）
- 修复拖拽后地图标记变为红色默认marker（补充iconPath和anchor属性）
- 修复最近邻排序对2个景点不生效（条件从<=2改为<=1）

### 交互调整

- 盲游/半盲模式下隐藏"添加日程项"按钮和拖拽功能
- 全明模式下显示"长按拖动调整顺序"提示文字
- 拖拽时禁用scroll-view滚动防止手势冲突

### 后端

- 新增批量线路景点查询接口（selectAllByLineIds）
- 行程生成器引入GenerationContext跨天状态管理
- 景点重置顺序改用replaceDayTouristAttractionIds写入NULL（替代空字符串）
- 盲游模式数据层：中间表、域对象、Mapper
- 盲游核心服务：解锁/跳过/强制解锁 + GPS校验
- 盲游模式完整测试套件（68个测试方法）

## 2026-06-15

### 功能更新

- 首页左上角用户头像改为"创建行程"按钮（深绿圆形 + 日历加号图标）
- 行程详情所有卡片（景点、住宿、餐饮、包车、跟拍）新增删除功能：
  - 删除前弹出确认弹窗，防止误操作
  - 景点删除：过滤掉目标 ID 后重新提交列表
  - 住宿/包车/跟拍/餐饮：调用后端新增的专用删除接口
- 行程详情按钮功能实现：
  - 景点导航按钮：调用 `wx.openLocation` 唤起微信导航
  - 住宿预订按钮：调用 `wx.makePhoneCall` 拨打联系电话
  - 餐饮预订按钮：调用 `wx.openLocation` 导航到餐厅
  - 包车/跟拍联系按钮：调用 `wx.makePhoneCall` 拨打电话
- 行程详情数据字典值转换：
  - 休闲指数：0→休闲、3→中等、5→困难
  - 不宜出行因素：0→高温、1→暴雨、2→冰冻、3→心脏病、4→高血压、5→恐高（支持多值）
  - 住宿类型：0→酒店、1→民宿
  - 含早餐/宠物友好：兼容 `'Y'` 和 `'1'`

### 界面中文化

- 卡片标签：STAY→住宿、BREAKFAST/LUNCH/DINNER→早餐/午餐/晚餐、CHARTERED→包车、PHOTO→跟拍
- 详情抽屉 section 标题全部汉化（LOCATION→位置、ABOUT→简介、DETAILS→详情等）
- 按钮文案：Add to Itinerary→添加到行程、Book Stay→预订住宿、Book Table→预订餐位、Contact Driver→联系司机
- 图片标签/空状态：Scenic Spot→景点、Local Stay→住宿、Local Flavor→地道美食等
- 其他：Photos→张、/person→/人、The Itinerary Path→行程路线

### 图标资源

- 新增并上传至 OSS：
  - `icon-create-itinerary.svg`（创建行程按钮）
  - `icon-phone.svg`（电话/预订按钮）
  - `icon-navigate.svg`（导航按钮）
  - `icon-add-itinerary.svg`（添加到行程按钮）
- 所有详情抽屉按钮图标从文字符号改为 SVG image

### 后端接口

- 新增 4 个 Wx 端删除接口：
  - `POST /wx/itinerary/{id}/day/{dayNumber}/accommodation/remove`
  - `POST /wx/itinerary/{id}/day/{dayNumber}/dining/remove?meal=xxx`
  - `POST /wx/itinerary/{id}/day/{dayNumber}/car/remove`
  - `POST /wx/itinerary/{id}/day/{dayNumber}/photography/remove`
- Service 层新增 `removeDayDining` 方法，只清除指定餐次不影响其他两餐

### 问题修复

- 修复餐饮删除会误清其他餐次的严重 bug（原 updateDayDining 会同时设置三个字段为 null）
- 修复 `attractionId` 类型不一致导致景点删除过滤失效（改用 String 比较）
- 修复 `wx.makePhoneCall` 失败时无反馈（拨号失败现显示号码提示）
- 修复餐饮删除 meal 参数缺失时静默空请求（添加防御性校验）
- 去除 `as any` 类型绕过，所有删除改用专用接口

## 2026-05-27

### 功能更新

- 行程添加抽屉新增关键词搜索能力：
